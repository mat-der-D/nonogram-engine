# Web アプリの仕様変更について

## 概要

[blue-print.md](./blue-print.md) および [repository-plan.md](./repository-plan.md) では、Web アプリの機能としてシンプルなソルバ機能およびその周辺機能を仕様として設定していた。

今回、アプリの目的を「Nonogram の問題を作ること」に絞り、以下の機能を提供するよう変更する。

- ベースは白黒のドット絵エディタとする。ドット絵作成のための基本的な機能を揃える。
- 支援機能として、画像からドット絵を作成する機能を与える。[](./figure_converter_spec.md)にある仕様で、下絵からドット絵を読み込むことができるようにする。
- ユーザーは上記機能で読み込んだ画像を引き続きエディア機能で編集できる。
- 絵が完成したと思ったら、ソルバー機能を起動して、解が一意に定まるか確認する。ダメそうなら、複数回の形を参考にしつつ修正する。
- 解が一意に定まるようになったら問題としてエクスポートする。JSONだけでなく、画像ファイルとしてのエクスポートもサポートする。

以上の仕様変更は Desktop App にも適用する。

## 技術要件

以下の機能を新たに nonogram-core, nonogram-wasm で提供する:

- 画像ファイルおよび各種パラメーターを与えると Grid を返す機能

これに伴って、Grid の json スキーマを nonogram-format に定義する。

apps/web/ では少なくとも以下の対応が必要:

- ユーザーの操作フローを再構築し、それに基づいて抜本的に画面レイアウトを再構成する
- 新機能(convert 機能, 画像 Export 機能)を実装する
