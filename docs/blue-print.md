# リポジトリ構想

## 概要

このリポジトリは お絵かきロジック(nonogram)のソルバ実装と、各種UIの実装を与える。

## 要件

- リポジトリはソルバ実装とUI実装の二層からなる。
- 全体は cargo の workspace を用いる。
- 開発には cc-sdd を用いる。

### ソルバの要件

- Rust で実装する。
- nonogramを解く問題はNP困難のため、決定版のアルゴリズムが存在しない。簡素で実装が軽いアルゴリズムの他、実装が複雑だが本格的なアルゴリズムも実装する。
- 各ソルバは trait を用いてポリモーフィズムを実現する
- 後からソルバを追加しやすい設計にする
- ソルバは部分的に解くのではなく、完全に解く(=解なし/唯一解/複数回の判定まで行う)仕様にする。
- バックトラッキング処理部分など、ソルバで共通して使える処理についてはなるべく抽象化して共通化する。
- 十分なドキュメント(American English)と単体テストで質を担保する。

### UIの要件

- 少なくとも以下をサポートする
  1. Web アプリケーション(GUI)
  2. デスクトップアプリケーション(GUI)
  3. CLI
- GUIについては以下の機能を提供する。
  - json 形式の問題インポート/エクスポート機能
  - json 形式の解答エクスポート機能
  - 問題入力機能
    - 列・行のヒントを順に入力する機能
    - グリッドを塗って列・行のヒントを生成する機能
  - 入力済みのヒントを元に問題を解く機能
    - 解答がない場合→解がない旨を表示する
    - ひとつの解の場合→一意解であること、およびその解答を表示する
    - 2つ以上解が存在する場合→解答のうち2つを表示する
- CLIについては以下の機能を提供する。
  - json 形式の問題を入力にして、解答の json を出力する機能
    - 引数でソルバのアルゴリズムを選択できるようにする
  - 問題のテンプレートを生成する機能
  - ヘルプ機能
- Webアプリケーションでは、WebAssemblyを用いて、バックエンドなしで動作するようにする。
- Webアプリケーションのフレームワークは軽量なSPAに適したものを採用する。
- デスクトップアプリケーションでは Tauri を用いる。
- GUIのフロントエンドはいずれも React+TypeScriptを採用する。
- GUI/CLIともに、json フォーマットは共通とする。
  - フォーマット管理用のクレートを独立させておいたほうがよいかも。
