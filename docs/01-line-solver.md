# DP ラインソルバ（Line Solver）

## 核心的アイディア

各行・列を**独立した1次元問題**として扱い、動的計画法でその行・列のすべての合法的配置を効率的に列挙する。「最左配置」と「最右配置」を求め、その**重複部分（オーバーラップ）**が必ず塗られるセルを確定する。

- 最左配置：すべてのブロックをできるだけ左に寄せた配置
- 最右配置：すべてのブロックをできるだけ右に寄せた配置
- 最左・最右の両方でブロックが重なるセル → **確定で塗り**
- どの合法的配置でもブロックが入らないセル → **確定で空白**

## アルゴリズムの特徴

- **問題の分解**：2次元グリッドを行・列の独立した1次元問題の集合として扱う
- **完全性**：DP テーブルにすべての合法的配置の情報を保持するため、論理的に確定できるセルをすべて確定できる
- **キャッシュとの親和性**：行の状態をキーとしてハッシュテーブルにキャッシュすると、困難なパズルでもヒット率 90% 超を達成できる
- **逐次適用**：行・列を交互に繰り返し適用し、確定セルが増えるごとに次の行・列で使える情報が増える（不動点まで繰り返す）

## メリット

- 実装が比較的シンプル
- 時間計算量が **O(n × k)**（n = 行長、k = クルー数）と効率的
  - 従来手法は O(n² × k) だったため大幅な改善
- 多くの易〜中程度のパズルはこれだけで完全に解ける
- ソルバ全体の基盤となり、他の手法と組み合わせやすい

## デメリット

- 行と列を**独立に**扱うため、行と列の組み合わせによって生まれる制約を直接利用できない
- 困難なパズルでは不動点（これ以上確定セルが増えない状態）に早期に達し、単独では解けない
- バックトラッキングなしでは解の存在を保証できない

## アルゴリズムの概要

### 前提

- 行の長さを `n`、クルー列を `[c_1, c_2, ..., c_k]` とする
- 各セルは `FILLED`・`EMPTY`・`UNKNOWN` の3値を持つ

### ステップ 1: 最左配置の計算

各ブロックをできるだけ左に詰めた配置を求める。

1. ブロック `i` の最左開始位置 `left[i]` を求める：
   - ブロック 1 の最左位置は 0
   - ブロック `i` の最左位置は `left[i-1] + c_{i-1} + 1`（直前ブロック長 + 最低1マスの空白）
2. すでに `EMPTY` が確定しているセルを考慮して、配置不可能な位置をスキップする

### ステップ 2: 最右配置の計算

各ブロックをできるだけ右に詰めた配置を求める（左右対称の処理）。

1. ブロック `i` の最右終了位置 `right[i]` を求める：
   - ブロック `k` の最右終了位置は `n - 1`
   - ブロック `i` の最右終了位置は `right[i+1] - c_{i+1} - 1`
2. すでに `FILLED` が確定しているセルを考慮して調整する

### ステップ 3: オーバーラップの確定

ブロック `i` について、最左開始位置と最右開始位置を比較する：

```
left[i] から left[i] + c_i - 1  (最左でブロックが占めるセル範囲)
right[i] から right[i] + c_i - 1 (最右でブロックが占めるセル範囲)
```

これらの範囲が重なる部分（`right[i]` から `left[i] + c_i - 1` まで）は、必ず `FILLED`。

### ステップ 4: 空白セルの確定

どのブロックの最左〜最右範囲にも含まれないセルは `EMPTY` に確定する。

### 全体ループ

```
repeat:
    changed = false
    for each row:
        apply line solver → update cells → if any change: changed = true
    for each column:
        apply line solver → update cells → if any change: changed = true
until not changed
```

不動点に達した時点で、解けていなければ後続の手法（アーク整合性・バックトラッキング）に委ねる。
