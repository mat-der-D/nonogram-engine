## タスク

- [ ] 1. 基盤セットアップ: 依存クレート追加・モジュール構成・エラー型定義
  - `nonogram-core`（path 依存）、`serde`（derive feature）、`serde_json`、`thiserror` を `Cargo.toml` に追加する
  - 空スタブ（`pub fn add`）を削除し、公開 API のモジュール構成を整える
  - JSON 変換操作の失敗を表現する共通エラー型（JSON エラー・パズル構築エラー・Unknown セルエラーの3バリアント）を定義する
  - `serde_json::Error` と `nonogram_core::Error` からの自動変換（`#[from]`）を設定する
  - _Requirements: 1.4, 1.5, 2.6_

- [ ] 2. 問題 JSON デシリアライズの実装とテスト
  - `row_clues` と `col_clues` を持つ内部 DTO 型を定義し、`serde` の derive マクロで JSON デシリアライズ可能にする
  - JSON 文字列を受け取り、DTO を経由して各クルーを構築し、`Puzzle` を組み立てて返す変換関数を実装する
  - 空配列 `[]` をゼロブロッククルーとして正しく扱えるようにする
  - フィールド欠落・`u32` 範囲超過は `serde_json` の標準検証で自動的にエラーとなることを活用する
  - 正常系テスト: 有効な JSON から `Puzzle` が正しく生成されること、空配列クルーの扱い、複数ブロッククルーの変換を検証する
  - 異常系テスト: `row_clues` 欠落、`col_clues` 欠落、`u32` 範囲超過でそれぞれエラーが返ることを検証する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 4.1, 4.3_

- [ ] 3. 解答 JSON シリアライズの実装とテスト
  - ステータス文字列と解答グリッド配列を持つ内部 DTO 型を定義し、`serde` の derive マクロで JSON シリアライズ可能にする
  - `SolveResult` の3バリアント（解なし・唯一解・複数解）をそれぞれ対応するステータス文字列にマッピングする変換関数を実装する
  - グリッドを行優先順で走査し、`Filled` → `true`、`Blank` → `false` に変換する
  - `Cell::Unknown` が含まれる場合は即座にエラーを返す（fail-fast）
  - 正常系テスト: `NoSolution`・`UniqueSolution`・`MultipleSolutions` それぞれの JSON 出力形式を検証する
  - 正常系テスト: グリッドのインデックス順が行優先であることを検証する
  - 異常系テスト: `Cell::Unknown` を含むグリッドでエラーが返ることを検証する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 2.6, 4.2, 4.3_

- [ ] 4. テンプレート生成・ラウンドトリップ検証・ワークスペース統合
  - 行数・列数を受け取り、全クルーを空配列に設定した問題テンプレート JSON を生成する関数を実装する
  - DTO 型をシリアライズに再利用し、指定サイズの空クルー配列を構築する
  - テスト: 指定サイズに応じた `row_clues`・`col_clues` の長さが正しいことを検証する
  - テスト: 生成された JSON がデシリアライズ関数でエラーなく変換できること（ラウンドトリップ）を検証する
  - テスト: 出力 JSON の構造が期待する形式と一致することを検証する
  - `cargo test --workspace` で全テストが通過することを確認する
  - _Requirements: 3.1, 3.2, 3.3, 4.4, 4.5_
