# 実装タスク

## タスク一覧

- [ ] 1. 開発環境セットアップと依存関係の設定
- [ ] 1.1 WASMパッケージのビルドとパッケージ参照を設定する
  - `wasm-pack build` で `nonogram-wasm` パッケージを生成する
  - `apps/web/package.json` に `"nonogram-wasm": "file:../../crates/nonogram-wasm/pkg"` を追加し `bun install` を実行する
  - _Requirements: 1.1, 1.2_

- [ ] 1.2 ViteのWASM ESMサポートを設定する
  - `vite-plugin-wasm` と `vite-plugin-top-level-await` を devDependencies に追加する
  - `apps/web/vite.config.ts` に両プラグインを登録する
  - _Requirements: 1.1, 1.2_

- [ ] 2. ユーティリティ層とサービス層の実装
- [ ] 2.1 (P) ヒント文字列パース・バリデーションユーティリティを実装する
  - カンマまたはスペース区切りの正整数列をパースして `number[]` に変換する機能を実装する
  - 空文字列を空配列（空ヒント）として有効なケースとして処理する
  - 正整数以外の文字（小数・負数・アルファベット等）を含む場合はエラー情報を返す
  - _Requirements: 2.3, 2.4, 2.5_

- [ ] 2.2 (P) グリッドセルからヒントを計算するユーティリティを実装する
  - `boolean[][]` グリッドを受け取り、各行の連続塗りつぶしセル数から行ヒントを計算する
  - 各列についても同様に列ヒントを計算する
  - 行または列が全て空白の場合は空配列を返す
  - _Requirements: 3.4_

- [ ] 2.3 (P) PuzzleIOServiceを実装する
  - `row_clues` と `col_clues` を含む問題JSONのファイルダウンロード機能を実装する
  - 行数・列数を受け取りヒントが空配列のテンプレートJSONをダウンロードする機能を実装する
  - 求解結果（`status` と `solutions` フィールド）の解答JSONをダウンロードする機能を実装する
  - JSONファイルをインポートしてスキーマ検証（`row_clues`・`col_clues` の存在と型）を行う機能を実装する
  - スキーマ不正時はエラー種別（parse/schema）と詳細メッセージを返す
  - Blob URL は使用後に `URL.revokeObjectURL()` で即座に解放する
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 6.2_

- [ ] 3. WASMプロバイダーの実装
- [ ] 3.1 WasmContextとWasmProviderを実装する
  - `loading` / `error` / `ready` の3状態を管理するContextを作成する
  - アプリマウント時に `nonogram-wasm` を動的インポートし `init()` で初期化する
  - 初期化成功時に `solve` 関数をコンテキスト経由で公開し、未完了時はエラーを返すノーオペレーション実装を提供する
  - 初期化失敗時にエラーメッセージを状態に格納し `console.error` でログ出力する
  - `useWasm()` カスタムフックを提供する
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [ ] 4. パズル状態管理の実装
- [ ] 4.1 useNonogramStoreの基本状態とディメンション管理を実装する
  - `useReducer` による単一ステートオブジェクト（行数・列数・ヒント入力・グリッドセル・入力モード・求解状態）を構築する
  - 行数・列数変更時にヒント入力配列とグリッドセル配列を正しいサイズに再初期化するアクションを実装する
  - `gridCells` が常に `rows × cols` の2D配列を維持する不変条件を守る
  - _Requirements: 2.1, 2.2_

- [ ] 4.2 ヒント入力とバリデーション管理を実装する
  - 各行・各列ヒントの入力文字列を更新するアクションを実装する
  - `clueParseUtils` を使ってリアルタイムにバリデーションし、エラー情報を `clueErrors` に反映する
  - 空ヒントを有効値として扱いエラーとしない
  - _Requirements: 2.3, 2.4, 2.5_

- [ ] 4.3 グリッド描画状態とモード切り替えを実装する
  - セルのクリックによるトグルアクションと、ドラッグ開始・移動・終了アクションを実装する
  - グリッドセルが変更された際に `clueComputeUtils` でヒントをリアルタイム計算し `rowClueInputs`/`colClueInputs` を更新する
  - ヒント入力モードとグリッド描画モードを相互に切り替えるアクションを実装する
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5_

- [ ] 4.4 求解アクションと結果状態管理を実装する
  - `idle` → `solving` → `done` の求解フェーズ遷移を管理するアクションを実装する
  - `await new Promise(r => setTimeout(r, 0))` を挿入してReactがローディング状態を描画してからWASM `solve()` を呼び出す
  - 求解結果（`none` / `unique` / `multiple` / `error`）を状態に格納する
  - `loadPuzzle` と `getPuzzleJson` で問題のロードとJSON変換を実装する
  - _Requirements: 1.4, 5.1, 5.2, 5.3, 5.4, 5.5, 6.1, 6.3_

- [ ] 5. ヒント入力UIコンポーネントの実装
- [ ] 5.1 (P) PuzzleSizeInputコンポーネントを実装する
  - 行数と列数の数値入力フィールドを提供する
  - 1〜50の範囲外の入力にはバリデーションエラーを表示する
  - 確定操作でストアの `setDimensions()` を呼び出す
  - _Requirements: 2.1, 2.2_

- [ ] 5.2 (P) ClueInputPanelコンポーネントを実装する
  - 行ヒントと列ヒントの入力欄を行・列ごとに表示する
  - 各入力欄の変更でストアのヒント更新アクションを呼び出す
  - `clueErrors` に対応するエラーが存在する場合、入力欄下にインラインエラーメッセージを表示する
  - `inputMode === 'grid'` の場合は非表示にする
  - _Requirements: 2.3, 2.4, 2.5_

- [ ] 5.3 (P) ModeToggleコンポーネントを実装する
  - 「ヒント入力」と「グリッド描画」の2モードを切り替えるトグルボタンを提供する
  - 現在のモードを視覚的に示し、クリックでストアの `setInputMode()` を呼び出す
  - _Requirements: 3.5_

- [ ] 6. グリッド描画パネルの実装
- [ ] 6.1 GridDrawingPanelのグリッド表示とクリック操作を実装する
  - `rows × cols` のセルグリッドをノノグラム盤面形式（列ヒントを上部・行ヒントを左部に配置）で表示する
  - セルクリックでストアの `toggleCell()` を呼び出す
  - 各セルを `React.memo` でメモ化し再描画コストを最小化する
  - `inputMode === 'clue'` の場合は非表示にする
  - _Requirements: 3.1, 3.2, 3.4_

- [ ] 6.2 GridDrawingPanelのドラッグ操作を実装する
  - `onPointerDown` でドラッグ開始セルの操作種別（塗りつぶし/消去）を確定してローカル状態に保持する
  - `setPointerCapture()` でポインタをキャプチャしてグリッド外でも `onPointerUp` を確実に捕捉する
  - `onPointerEnter` でドラッグ中のすべての通過セルにドラッグ開始時の操作を適用する
  - ドラッグ中セルに視覚的なハイライトフィードバックを表示する
  - _Requirements: 3.3_

- [ ] 7. インポート/エクスポートパネルの実装
- [ ] 7.1 ImportExportPanelを実装する
  - 問題エクスポート・テンプレートエクスポートボタンを提供し、クリックで `PuzzleIOService` の対応メソッドを呼び出す
  - `<input type="file" accept=".json">` でファイル選択し `PuzzleIOService.importPuzzle()` を呼び出してストアに反映する
  - インポートエラー（パースエラー・スキーマエラー）をコンポーネントローカル状態でインラインメッセージとして表示する
  - _Requirements: 4.1, 4.2, 4.3, 4.4_

- [ ] 8. 求解・結果表示コンポーネントの実装
- [ ] 8.1 (P) SolveButtonコンポーネントを実装する
  - `solvePhase.phase === 'solving'` のときローディングインジケーターを表示してボタンを無効化する
  - WASMが `ready` かつ `clueErrors` が空の場合のみボタンをクリック可能にする
  - クリックでストアの `solve()` を呼び出す
  - _Requirements: 5.1, 5.2_

- [ ] 8.2 (P) SolutionGridコンポーネントを実装する
  - `boolean[][]` グリッドを塗りつぶしセルと空白セルを視覚的に区別して表示する
  - `label` が指定された場合はグリッド上部にラベルを表示する
  - セルサイズを結果パネル幅に合わせて自動調整する
  - _Requirements: 5.6_

- [ ] 8.3 ResultPanelを実装する
  - `solvePhase` の値に応じて解なし・唯一解・複数解・エラーの4状態の表示を切り替える
  - 唯一解時は「唯一解」ラベルとともに `SolutionGrid` を1つ表示する
  - 複数解時は2つの `SolutionGrid` を並べて表示し、「グリッドを描き直してください」誘導メッセージを添える
  - 解なし時は「解が存在しないパズルです」メッセージを表示する
  - 解が存在する（`unique` または `multiple`）ときのみ解答エクスポートボタンを有効化し、クリックで `PuzzleIOService.exportSolution()` を呼び出す
  - _Requirements: 5.3, 5.4, 5.5, 5.6, 6.1, 6.2, 6.3_

- [ ] 9. アプリケーション統合
- [ ] 9.1 Appコンポーネントと2カラムレイアウトを実装する
  - `WasmProvider` でアプリ全体をラップし、WASM初期化中はローディングスピナーを全面表示する
  - WASM初期化失敗時はエラーバナーを表示し求解関連コンポーネントを無効化する
  - 左右2カラムレイアウトを実装し、左パネルに入力UI・右パネルに結果を配置する
  - ヘッダーに `ImportExportPanel` を配置する
  - _Requirements: 1.3_

- [ ] 9.2 全コンポーネントの結合と動作確認を行う
  - ヒント入力→求解→結果表示（唯一解・複数解・解なし）の一連フローを動作確認する
  - グリッド描画モードでのセル描画→ヒント自動計算→求解→結果表示フローを動作確認する
  - 問題インポート→求解→解答エクスポートの一連フローを動作確認する
  - 求解中は左パネルの入力が無効化され、求解完了後に編集可能に戻ることを確認する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 2.2, 2.3, 2.4, 2.5, 3.1, 3.2, 3.3, 3.4, 3.5, 4.1, 4.2, 4.3, 4.4, 5.1, 5.2, 5.3, 5.4, 5.5, 5.6, 6.1, 6.2, 6.3_
