# 要件定義書

## はじめに

本ドキュメントはノノグラム（お絵かきロジック）Webアプリケーション（`apps/web`）の要件を定義する。
本アプリは Vite + React + TypeScript で構築し、`nonogram-wasm` を介してブラウザ内でノノグラムを完全求解するSPAである。バックエンドサーバーは不要で、すべての処理をクライアントサイドで完結させる。

---

## 要件

### 要件 1: アプリケーション基盤・WASM統合

**目的:** ユーザーとして、バックエンドなしでブラウザ上でノノグラムソルバを使いたい。そうすることでインストール不要かつオフラインでも利用できる。

#### 受け入れ基準

1. The Web App shall バックエンドサーバーへのリクエストなしに動作する。
2. When ページが読み込まれたとき、the Web App shall `nonogram-wasm` モジュールを初期化する。
3. If WASMモジュールの初期化に失敗したとき、the Web App shall エラーメッセージをユーザーに表示する。
4. The Web App shall `nonogram-wasm` の `solve(puzzle_json: &str) -> String` 関数を呼び出してパズルを解く。

---

### 要件 2: 問題入力（ヒント入力方式）

**目的:** ユーザーとして、行・列ヒントを順に入力して問題を定義したい。そうすることでJSONを手書きせずにパズルを作成できる。

#### 受け入れ基準

1. The Web App shall 行数と列数を指定する入力フィールドを提供する。
2. When ユーザーが行数と列数を確定したとき、the Web App shall 各行のヒント入力欄と各列のヒント入力欄を生成して表示する。
3. When ユーザーがヒントを入力したとき、the Web App shall カンマ区切りまたはスペース区切りの正整数列として各ヒントを受け付ける。
4. If ヒント入力値が正整数以外の文字を含む場合、the Web App shall 該当入力欄にバリデーションエラーを表示する。
5. The Web App shall 各行・各列に空のヒント（ヒントなし＝すべて空白）を入力可能にする。

---

### 要件 3: 問題入力（グリッド描画方式）

**目的:** ユーザーとして、グリッドのセルを塗りつぶすことでパズルのヒントを自動生成したい。そうすることでビジュアルに問題を作成できる。

#### 受け入れ基準

1. The Web App shall グリッド描画モードを提供する。
2. When ユーザーがグリッドのセルをクリックしたとき、the Web App shall そのセルの塗りつぶし状態をトグルする。
3. When ユーザーがグリッド上をドラッグしたとき、the Web App shall ドラッグ開始時の操作（塗りつぶし or 消去）に応じてすべての通過セルに同じ操作を適用する。
4. While グリッド描画モードがアクティブなとき、the Web App shall 塗りつぶし状態に基づいて行・列ヒントをリアルタイムに自動計算して表示する。
5. The Web App shall グリッド描画モードとヒント入力モードを相互に切り替え可能にする。

---

### 要件 4: 問題インポート / エクスポート

**目的:** ユーザーとして、JSON形式でパズルをインポート・エクスポートしたい。そうすることで問題を保存・共有できる。

#### 受け入れ基準

1. When ユーザーが問題エクスポートを実行したとき、the Web App shall `docs/repository-plan.md` セクション4.1 で定義されたJSON形式（`row_clues`・`col_clues` フィールド）のファイルをダウンロードさせる。
2. When ユーザーが問題JSONファイルをインポートしたとき、the Web App shall そのファイルを解析して現在の問題入力フィールドに反映する。
3. If インポートされたJSONが定義済みスキーマ（`row_clues`・`col_clues` が必須かつ配列）に適合しない場合、the Web App shall エラーメッセージを表示してインポートを中断する。
4. Where 問題テンプレート生成機能が含まれる場合、the Web App shall 行数・列数を指定して全ヒントを空配列にしたテンプレートJSONをダウンロードさせる。

---

### 要件 5: 問題求解・結果表示

**目的:** ユーザーとして、入力済みヒントを元にパズルを解き、結果を視覚的に確認したい。そうすることでノノグラムを手動で解く必要がなくなる。

#### 受け入れ基準

1. When ユーザーが「解く」ボタンを押したとき、the Web App shall `nonogram-wasm` を通じて現在の問題を求解する。
2. While 求解処理が実行中のとき、the Web App shall ローディングインジケーターを表示する。
3. When 求解結果が `"none"`（解なし）のとき、the Web App shall 「解が存在しない」旨のメッセージを表示する。
4. When 求解結果が `"unique"`（唯一解）のとき、the Web App shall 一意解であることを示すラベルとともに解答グリッドを1つ表示する。
5. When 求解結果が `"multiple"`（複数解）のとき、the Web App shall 2つの解答例をグリッドで並べて表示する。
6. The Web App shall 解答グリッドのセルを塗りつぶし状態（塗りつぶし / 空白）に応じて視覚的に区別して表示する。

---

### 要件 6: 解答エクスポート

**目的:** ユーザーとして、求解結果をJSON形式でエクスポートしたい。そうすることで解答データを他の用途に再利用できる。

#### 受け入れ基準

1. When 求解が完了して結果が表示されているとき、the Web App shall 解答エクスポートボタンを有効化する。
2. When ユーザーが解答エクスポートを実行したとき、the Web App shall `docs/repository-plan.md` セクション4.2 で定義されたJSON形式（`status`・`solutions` フィールド）のファイルをダウンロードさせる。
3. While 解答が存在しない状態（未求解または求解結果が解なし）のとき、the Web App shall 解答エクスポートボタンを無効化する。
